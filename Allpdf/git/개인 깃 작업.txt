1.폴더 생성(이 폴더 안의 파일들을 형상관리 하겠다)
2.폴더 안에서 git 실행 -> git init 명령어 실행
    .git 이라는 숨김폴더 생성됨
    .git 폴더가 생성된 폴더의 파일들은 형상관리를 할 수 있음.
    형상관리가 가능한 폴더를 지역저장소(local repository)라 부른다.
3. 폴더에 형상관리할 파일을 생성, 추가
4. git add 파일명 or git add . 명령어로 새로 추가한 파일을 형상관리 시작할 수 있다.
5. 파일 작업한다(수정)
6. 작업한 내용의 백업 위치를 지정한다. git commit -m "커밋메세지"
7. 작업한 파일을 gitHub에 올리기 위해 remote repository 생성
8. 지역저장서와 원격저장소를 연결(최초 한 번만)
    git remote add origin 원격저장소 url
9. 지역 저장소의 파일을 원격저장소에 전송
     git push origin main

--------------------------------------
한 번 github에 파일을 올리면 아래 작업을 반복
1. 지역저장소의 파일을 수정하거나, 추가하거나, 삭제한다.(할 거 한다)
2. 작업한 내역을 staging 한다
     git add .
3. 작업한 내용의 복원시점을 기록한다.
     git commit -m "커밋메세지"
4. 작업한 내용을 원격저장소로 전송한다.
    git push origin main
----------------------------------------
원격저장소의 내용을 다른 곳에서 내려 받기
1. 내려받을 폴더에서 git bash 실행
2. 내려받는 명령어 사용
    git clone 원격저장소url
------------------------------------------
강의실, 집에 모두 연결이 된 상태에서 작업 방법
0. 다른 곳에서 작업한 내용 내려받기
     git pull origin main
1. 지역저장소의 파일을 수정하거나, 추가하거나, 삭제한다.(할 거 한다)
2. 작업한 내역을 staging 한다
     git add .
3. 작업한 내용의 복원시점을 기록한다.
     git commit -m "커밋메세지"
4. 작업한 내용을 원격저장소로 전송한다.
    git push origin main
